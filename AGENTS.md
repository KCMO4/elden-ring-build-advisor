### .sl2 reading ‚Äî critical points

The parser reverse-engineers FromSoftware's BND4 format:

- **Slot size**: `0x280010` bytes per slot; slot data starts at `offset 0x310 + slot * 0x280010`
- **Stats**: `findStats()` locates the 8 attributes by character level (signature pattern)
- **ChrAsm2** (equipped gear) = `vigor_offset + 0x310`, 96 bytes:
  - `+0x00..+0x17` ‚Üí LH[0..2] / RH[0..2] (interleaved: LH1,RH1,LH2,RH2,LH3,RH3)
  - `+0x18..+0x27` ‚Üí arrows + bolts
  - `+0x28..+0x37` ‚Üí 4 unknown fields (note: ER-Save-Editor Rust struct mislabels these)
  - `+0x38..+0x44` ‚Üí HEAD, CHEST, ARMS, LEGS
  - `+0x48` ‚Üí unk; `+0x4C..+0x58` ‚Üí talisman[0..3]
- **gaitem_handle** ‚Äî high byte indicates category:
  - `0x80xxxxxx` ‚Üí weapon
  - `0x90xxxxxx` ‚Üí armor
  - `0xA0xxxxxx` ‚Üí talisman
- **Weapon IDs**: `baseId = floor(item_id / 100) * 100`, `upgradeLevel = item_id % 100`
- **Armor IDs**: `baseId = item_id XOR 0x10000000`
- **Talisman IDs**: `baseId = handle XOR 0xA0000000`

### EquippedWeapon ‚Äî available fields

```typescript
interface EquippedWeapon {
  rawId: number;           // raw gaitem_handle from .sl2
  baseId: number;          // base item ID (without upgrade level or infusion)
  name: string | null;     // null if not found in database
  upgradeLevel?: number;   // +0 to +25 / +10 (weapons only)
  image?: string;          // fanapis URL (may be empty)
  infusion?: string;       // "Heavy" | "Keen" | "Fire" | ‚Ä¶ (infused weapons only)
  damage?: DamageStats;    // weapons only
  scaling?: ScalingStats;  // weapons only
  weight?: number;         // weapons, armors, shields
  defense?: DefenseStats;  // armors only (float precision for 550/568 items)
  poise?: number;          // armors only (from EldenRingArmorOptimizer)
  immunity?: number;       // armors only ‚Äî resistance to Poison/Scarlet Rot
  robustness?: number;     // armors only ‚Äî resistance to Hemorrhage/Frostbite
  focus?: number;          // armors only ‚Äî resistance to Sleep/Madness
  vitality?: number;       // armors only ‚Äî resistance to Death Blight
  stability?: number;      // shields only ‚Äî Guard Boost value
  skill?: string;          // weapons only ‚Äî Ash of War skill name (resolved from ga_items)
  effect?: string;         // effect description (mainly talismans)
}
```

### Armor data precision

`armors.json` has two layers:
- **fanapis** (base): integer defense values, no poise
- **patch-armor** overlay: float precision from `jerpdoesgames/EldenRingArmorOptimizer`
  (extracted from game's `regulation.bin ‚Üí EquipParamProtector`)

Coverage after patching:
- **550 / 568** armors have float values + poise + resistances (e.g. `strike: 5.4, poise: 11, immunity: 28`)
- **18 unmatched**: 6 DLC Brave's set (optimizer predates Shadow of the Erdtree) + 12 obscure
  items with near-zero stats ‚Äî these keep integer approximations from fanapis

The `Armor` type includes `poise: number` (0 when unavailable) and optional `immunity`, `robustness`, `focus`, `vitality`.

### fanapis ‚Äî field name gotchas

The fanapis API uses non-standard names:
- `"Phy"` instead of `"physical"`
- `"Ligt"` instead of `"lightning"` (**typo in the API**)
- The `dmgNegation` field is an array of `{ name, amount }` ‚Äî use `getDef(key)` case-insensitively

### Talisman effects

**Two layers of data:**

1. **`utils/talismanEffects.ts`** ‚Äî `TALISMAN_EFFECTS` map with ~35 talismans that affect stats
   (Erdtree's Favor, Radagon's Soreseal, Arsenal Charm, Scorpion Charms, Dragoncrest,
   Graven-School/Mass, Shard of Alexander, etc.) with precise numeric values.
   - `getTalismanEffectLines(baseId)` ‚Üí `{ label, value }[]` for tooltips (e.g. `HP +3%`)
   - `computeTalismanBonuses(talismans)` ‚Üí aggregate bonuses for HP/FP/Stamina/Load/elemental dmg/defense
   - These values are applied in `DerivedStatsPanel` to compute real HP/FP/Stamina/Load and elemental AR bonuses

2. **`backend: EquippedWeapon.effect`** ‚Äî fanapis description text for **all** talismans.
   When `getTalismanEffectLines` returns `null`, the tooltip shows this text instead.

### Derived stats formulas (DerivedStatsPanel)

Implemented with exact piecewise interpolation (reverse-engineered by the community):

```
HP (Vigor):
  v ‚â§ 25 ‚Üí 300  + 500 * ((v-1)/24)^1.5
  v ‚â§ 40 ‚Üí 800  + 650 * ((v-25)/15)^1.1
  v ‚â§ 60 ‚Üí 1450 + 450 * (1 - (1-(v-40)/20)^1.2)
  v ‚â§ 99 ‚Üí 1900 + 200 * (1 - (1-(v-60)/39)^1.2)

FP (Mind):
  m ‚â§ 15 ‚Üí 50  + 45  * ((m-1)/14)
  m ‚â§ 35 ‚Üí 95  + 105 * ((m-15)/20)
  m ‚â§ 60 ‚Üí 200 + 150 * (1 - (1-(m-35)/25)^1.2)
  m ‚â§ 99 ‚Üí 350 + 100 * ((m-60)/39)

Stamina (Endurance):
  e ‚â§ 15 ‚Üí 80  + 25 * ((e-1)/14)
  e ‚â§ 30 ‚Üí 105 + 25 * ((e-15)/15)
  e ‚â§ 50 ‚Üí 130 + 25 * ((e-30)/20)
  e ‚â§ 99 ‚Üí 155 + 15 * ((e-50)/49)

Max Equip Load (Endurance):
  e ‚â§ 8  ‚Üí 45
  e ‚â§ 25 ‚Üí 45  + 27 * ((e-8)/17)
  e ‚â§ 60 ‚Üí 72  + 48 * ((e-25)/35)^1.1
  e ‚â§ 99 ‚Üí 120 + 40 * ((e-60)/39)

Poise:
  totalPoise = sum(head.poise, chest.poise, hands.poise, legs.poise)  (simple additive)

Resistances (Immunity, Robustness, Focus, Vitality):
  total = floor(levelComponent + attrComponent) + armorSum

  Level component (same for all 4, using internal lvl = runeLevel + 79):
    lvl ‚â§ 149 ‚Üí 75  + 30 * ((lvl-1)/149)
    lvl ‚â§ 190 ‚Üí 105 + 40 * ((lvl-150)/40)
    lvl ‚â§ 240 ‚Üí 145 + 15 * ((lvl-190)/50)
    lvl > 240 ‚Üí 160 + 20 * ((lvl-240)/552)

  Attr component (Immunity‚ÜêVIG, Robustness‚ÜêEND, Focus‚ÜêMND):
    stat ‚â§ 30 ‚Üí 0
    stat ‚â§ 40 ‚Üí 30 * ((stat-30)/10)
    stat ‚â§ 60 ‚Üí 30 + 10 * ((stat-40)/20)
    stat ‚â§ 99 ‚Üí 40 + 10 * ((stat-60)/39)

  Attr component (Vitality‚ÜêARC ‚Äî different curve):
    arc ‚â§ 15 ‚Üí arc
    arc ‚â§ 40 ‚Üí 15 + 15 * ((arc-15)/25)
    arc ‚â§ 60 ‚Üí 30 + 10 * ((arc-40)/20)
    arc ‚â§ 99 ‚Üí 40 + 10 * ((arc-60)/39)
```

### Animations

- **Bars**: `useMountAnimation()` hook ‚Äî triggers CSS transition after 1 rAF.
  Curve: `cubic-bezier(0.16, 1, 0.3, 1)` (easeOutExpo), duration 0.9s.
  Stagger: `transitionDelay: index * 60ms` per bar.
- **Page entry**: `fadeSlideDown` (header), `fadeSlideUp` (sidebar/content)
- **Character name**: `nameGlow` pulse every 4s
- **ItemSlot**: `slotReveal` (scale 0.88‚Üí1 + fade), 45ms stagger per slot
- **Hover ItemSlot**: diagonal shimmer sweep (`::after`) + gold rim light (`::before`)
- **UploadPage title**: `titleReveal` (letter-spacing collapses 0.5em‚Üí0.15em)
- **Spinner**: square (no border-radius), 1.2s‚Äì1.4s linear rotation

## Conventions

- **Strict TypeScript**: no `any`, prefer explicit types
- **CSS Modules**: each component has its `.module.css`; no inline styles except dynamic values (animations, bar colors)
- **No positive `border-radius`** on bars, panels, slots or badges
- **English** in all UI strings, comments, variable names and server logs
- **Do not manually edit `src/data/*.json`** ‚Äî always regenerate with `npm run sync-data`
- **Multer field name**: the multipart form field must be `savefile` (not `save` or `file`)
- **Do not commit loose `.js` files** in `frontend/src/` ‚Äî those are build artifacts. Only `.ts` and `.tsx`
